# Cross-lingual representation alignment — results

MEXA-style alignment for all 107 checkpoints (the full matched-token-budget
roster — `-1b`/`-2b`/`-5b`/`-8b`/`-10b`/`-12b`/`-15b`/`-23b` intermediate
checkpoints alongside the original 26 30B finals) on FLORES+ dev+devtest
(n=2009), all 10 language pairs, all 17 layers. Generated by
`scripts/external_bench/run_alignment.py` (fanned out via
`run_align_fanout2.sh`), exported with `scripts/external_bench/export_alignment.py`.
Method and full discussion: [CLAUDE.md §6b](../../CLAUDE.md) — note CLAUDE.md's
prose there still describes the original 26-model sweep; the CSVs here are
ahead of that write-up and the matched-token-budget analysis is not yet done.

Embeddings were not cached for this 107-model sweep (only the original 26 have
`.npz` files on scratch); the raw per-run JSONs (~130 MB, mostly per-query
bootstrap arrays) are **not** committed — they live on scratch at
`/mnt/scratch/xscript_c5/results/alignment/`. Everything needed to plot is here.

## Files

| file | rows | contents |
|---|---|---|
| `per_layer.csv` | 36380 | depth profiles: every metric at every layer |
| `summary.csv` | 4280 | the two reported layers (`ref`, `best`), incl. CKA |
| `lexical_floor.csv` | 20 | model-free TF-IDF floor, per tokenizer |

Key columns: `variant` (`raw` / `centered`), `layer` (0 = embedding output,
1..16 = block outputs), `n_trained_in_pair` (0/1/2 — how many of the pair's
languages the model actually trained on), `is_ref_layer`.

## Read this before plotting

**1. Always plot the lexical floor.** `lexical_floor.csv` is TF-IDF retrieval
over shared token ids with *no model at all*. FLORES leaks across scripts
(digits, dates, Latin-script names survive translation), so the floor is already
0.43 EN-DE / 0.58 EN-FR. Cells at or below it carry no representational signal.
The floor is **tokenizer-dependent**, so use the row matching each model's `tok`.

**2. `top1_*` saturates; prefer `dprime`.** Bidirectional top-1 exceeds 0.95 even
for monolingual *controls*, and every bilingual lands in 0.966–0.995, so
differences there are ceiling effects, not signal. `dprime`
(`(matched − mean_nonmatched) / std_nonmatched`, per query) is unbounded and
scale-free. `cosine_margin` and `cka` also don't ceiling.

**3. Do not compare models at a single fixed layer.** Alignment emerges at
different depths — bilinguals peak ~L15-16, monolinguals ~L12-16, and
starved-tokenizer models later still (`fr-starved` reaches its plateau ~8 layers
after `fr-fair`). Reading everything at `ref` (75% depth) produced a *sign-flipped*
same-script result that had to be retracted. **Plot the whole profile**; if you
must reduce to one number, report fixed-layer and peak-layer side by side.
`per_layer.csv` exists precisely so this is visible.

**4. Monolingual controls are the baseline, not the bilinguals.** An
Arabic-only model retrieves EN↔AR at 0.963 and a Chinese-only model EN↔ZH at
0.965 — neither trained on English. Raw per-model scores are close to
meaningless; only differences against a matched control mean anything. Use
`n_trained_in_pair` to separate in-domain (2), half-OOD (1) and fully-OOD (0)
cells.

**5. "Monolingual" checkpoints are not monolingual.** `ar-fair` scores 0.909 on
EN-FR and 0.802 on FR-AR (Maghreb web text is heavily French-Arabic bilingual);
English-only models reach 0.906 on EN-ZH. Incidental exposure inside
FineWeb2-HQ's per-language subsets is doing real work.

## Suggested plots

- **Depth profile**: `dprime` vs `layer`, one line per model, faceted by pair.
  The single most informative view — it shows the emergence-depth shift that
  every single-layer summary hides.
- **Transfer tiers**: `dprime` at peak layer, grouped by `n_trained_in_pair`
  (on the original 26-model sweep: 2 → 8.45, 1 → 4.11, 0 → 2.50 on average —
  recompute against the full 107 before quoting, these numbers predate that
  export).
- **Script effect on OOD pairs**: among `n_trained_in_pair == 0`, split by
  `same_script` (26-model sweep: 3.54 vs 2.12 — same caveat).
- **Matched-token-budget transfer curve** (new, needs the 107-model data):
  `dprime` at peak layer vs. token budget (`-1b` through `-23b` and the 30B
  final), same-script vs. cross-script partners, mirroring
  `bootstrap_transfer.py`'s matched-budget downstream analysis but for
  representation alignment instead of accuracy.
- **Model × pair heatmap**: `summary.csv`, `variant == centered`,
  `which_layer == best`, with the floor subtracted.
